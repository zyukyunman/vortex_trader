# 00 · 总览与定位

> 注：配置/端口以 vortex_common ADR-003 + config/registry.yml 为准（本文为历史记录）。

## 平台愿景

Vortex 是一套**面向中国 A 股的端到端量化平台**，把"数据 → 研究 → 回测 → 实盘"打通成一条可复现、可审计、可自动化的流水线。整个平台拆成四个职责单一、可独立部署的服务，通过稳定的接口契约协作，而非揉成一个巨石应用。

平台的最终交付物有两类：

1. **技术文档**——讲清整体设计、各服务接口协议、镜像运行与部署方式（即本套 `design/` 文档）。
2. **持续的量化策略研究产出**——以"研究院／交易员"的方式，源源不断地提出策略假设、用数据与回测验证、在风控下走向纸上与实盘。

## 四个服务的定位

- **vortex_data（数据服务）**：唯一的数据真相源。负责从 Tushare 采集 A 股行情、财务、事件等 50+ 数据集，落盘为 Parquet，并导出为 Qlib 兼容格式供回测与策略消费。自带采集调度、质量门禁、磁盘自卫、Web 看板。
- **vortex_backtest（回测引擎）**：消费 vortex_data 的数据，提供分钟级 A 股回测。内建 A 股微结构规则（T+1、分板手数、涨跌停、停牌、费用），异步作业模型 + 日级报告 + 策略看板。
- **vortex_qmt（实盘交易）**：把策略冻结的目标组合，经事前风控、QMT 桥接送入国金模拟盘/真实账户，并做日终对账。fail-closed 安全模型 + 实盘三重门禁 + 幂等执行。
- **vortex_trader（本库，研究院/枢纽）**：见下。

## 本库（vortex_trader）的定位

vortex_trader 不是第四个"数据/回测/交易"服务，而是平台的**大脑与门面**，承担三件事：

### 1. 文档枢纽
把分散在三个服务 `design/` 里的设计、接口、部署知识，**汇总成一套面向全局的技术文档**：整体架构（01）、接口协议（02）、镜像部署（03）。新成员或外部读者从本库即可建立全貌，再深入各服务。

### 2. 研究院（持续自主研究）
以"量化策略研究员"的身份运行一条**研究闭环**：提出假设 → 在 vortex_data 确认数据 → 构建因子/信号 → 提交 vortex_backtest 回测 → 评估稳健性 → 在 vortex_qmt 纸上/模拟验证执行 → 风控下实盘。研究方法论、agent 研究员分工、数据资产盘点与方向建议见 04。

### 3. 编排桥（策略 → 实盘）
当前平台存在一处明显空白：**回测/策略的产出，没有任何模块自动转成 vortex_qmt 的 `/rebalance` 目标组合**。这层"策略→目标组合"的编排，正是 vortex_trader 应当承担的交易员职责（见 01 集成缺口、04 路线图）。

## 职责边界（本库不做的事）

- **不重复实现数据/回测/交易逻辑**：取数走 vortex_data API/Qlib，回测走 vortex_backtest API，下单走 vortex_qmt API。本库只做编排与研究记录。
- **不绕过风控**：实盘一律经 vortex_qmt 的 fail-closed 风控与三重门禁，本库不直连券商。
- **不持有密钥**：Tushare token、桥接 token、写接口 token 等由各服务自管，本库通过接口协作。

## 阅读顺序

新读者建议：本文档（00）→ 端到端架构（01）→ 按需查接口（02）/ 部署（03）→ 研究方法论（04）。
