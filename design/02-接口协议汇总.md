# 02 · 接口协议汇总

本文件把三个服务的对外接口汇总到一处，便于编排与联调。**权威定义仍以各服务自身的 `design/` 与源码为准**（文末列出指引）。

## 通用约定

- **传输**：均为 HTTP/JSON（FastAPI / 内建 HTTP）。
- **鉴权模型（三服务一致的安全基线）**：
  - **只读接口**：无需鉴权。
  - **写接口**：配置了 token 时需 `X-API-Token: <token>` 或 `Authorization: Bearer <token>`（常量时间比较）；**未配置 token 时仅本机回环 `127.0.0.1/::1` 可写**，远程一律拒绝（fail-closed）。
  - **默认只绑回环**：compose 默认 `BIND_ADDR=127.0.0.1`，对外暴露前必须先配 token 再改 `0.0.0.0`。
  - ⚠️ Docker 网桥下宿主机请求源 IP 非回环，故 **Docker 部署若用写接口必须配置 token**。

| 服务 | 端口（内外一致） | 写鉴权 | 健康检查 |
|---|---|---|---|
| vortex_data | 8765 | token / 回环 | `GET /api/health` |
| vortex_backtest | 8766 | token（守卫待实现）/ 回环 | `GET /health` |
| vortex_qmt | 8767 | token / 回环 + 实盘三重门禁 | `GET /health` |

> 端口规范权威来源：vortex_common 的 `config/registry.yml` + [ADR-003](../../vortex_common/docs/adr/ADR-003-unified-config-architecture.md)。

## 一、vortex_data（数据服务）· http://127.0.0.1:8765

### 只读接口

| 方法 | 端点 | 说明 |
|---|---|---|
| GET | `/api/health` | 健康与 workspace |
| GET | `/api/datasets` | 数据集目录及状态（rows/status） |
| GET | `/api/datasets/{dataset}/schema` | 字段信息 |
| GET | `/api/datasets/{dataset}/sample?limit=20` | 样例数据 |
| GET | `/api/datasets/{dataset}/rows?symbol=000001.SZ&start=20240101&end=20240131` | 范围查询，支持 `columns=`、`limit=`、`format=csv` |
| GET | `/api/tasks` | 任务列表与进度 |
| GET | `/api/settings` | 当前配置（脱敏） |
| GET | `/api/storage` | 磁盘状态（free_gb / threshold） |
| GET | `/` | 看板 HTML |

### 写接口（token / 回环）

| 方法 | 端点 | 请求体要点 |
|---|---|---|
| POST | `/api/scheduler/pause` · `/resume` | 暂停/恢复自动调度 |
| POST | `/api/data/update-now` | `{datasets:[...], frequencies:["day"]}` 立即增量 |
| POST | `/api/data/task` | `{action:"bootstrap\|update\|backfill\|repair", datasets, start, end}` |
| POST | `/api/data/task/{id}/cancel` | 取消任务 |
| POST | `/api/full-capture/reset` | 重置全量计划 |
| PATCH | `/api/settings` | `{min_free_gb, scheduler_enabled, ...}` |
| PATCH | `/api/data/config` | `{workday_datasets, weekend_enabled, history_start, ...}` |

### CLI（`vortex-data`）

```bash
vortex-data init --root ./workspace
vortex-data server start --host 127.0.0.1 --port 8765   # status / stop
vortex-data data bootstrap|update|backfill|repair|status|logs|cancel|inspect
vortex-data export qlib --freq day --symbols 000001.SZ,600000.SH --start 20240101 --end 20240630 --out ./qlib_cn_day [--update]
vortex-data settings set min_free_gb=10 scheduler_enabled=false
```

### Qlib 导出契约（data → backtest 的核心接缝）

```
workspace/qlib/cn_day/    (cn_1min 结构相同，日级常量字段广播到 240 分钟 bar)
├── calendars/day.txt          # YYYY-MM-DD 逐行
├── instruments/all.txt        # SYMBOL\tstart\tend（大写 Qlib 代码）
└── features/sh600000/         # 实例目录小写
    ├── close.day.bin          # close 为 raw 未复权
    ├── factor.day.bin         # factor = adj / 全历史最新 adj（不随窗口漂移）
    ├── limit_up.day.bin / limit_down.day.bin / paused.day.bin / ...
```

字段集（12）：`open high low close volume amount vwap factor change paused limit_up limit_down`。复权价 `close_qfq = close * factor`；停牌日 `close=NaN`。

## 二、vortex_backtest（回测引擎）· http://127.0.0.1:8766

### 端点

| 方法 | 端点 | 说明 |
|---|---|---|
| GET | `/health` | 健康 |
| POST | `/accounts` | 建账户 `{account_id, initial_cash, engine?, name?}` |
| GET | `/accounts` · `/accounts/{id}` | 列/查账户 |
| POST | `/accounts/{id}/orders` | 下单（写） |
| GET | `/accounts/{id}/orders?order_batch_id=&start_date=&end_date=` | 列订单 |
| POST | `/backtests` | 提交回测，返回 `202 {job_id, status:"queued"}` |
| GET | `/backtests` · `/backtests/{job_id}` | 列/查作业（状态+进度） |
| GET | `/backtests/{job_id}/summary` · `/daily` · `/trades` · `/rejections` | 日级报告 |
| GET | `/accounts/{id}/summary` · `/positions` | 账户最近回测汇总/持仓 |
| GET | `/symbols/{symbol}` | 标的元数据（板块/最小手数） |

### 提交回测（请求）

```json
{
  "account_id": "demo",
  "frequency": "1min",
  "price_adjustment": "qfq",
  "start_date": "2026-01-02",
  "end_date": "2026-01-05",
  "strategies": [
    {"strategy_id": "main", "strategy_type": "order_replay",
     "initial_cash": 100000, "symbols": ["000001.SZ"],
     "params": {"order_batch_id": "batch-main"}}
  ],
  "execution": {"commission_rate": 0.0003, "min_commission": 5.0,
                "stamp_tax_rate": 0.0005, "transfer_fee_rate": 0.00001,
                "max_volume_participation": 1.0, "slippage_bps": 0.0}
}
```

- `frequency` 当前仅 `1min`；`price_adjustment` 仅 `qfq`。
- **多策略 = 独立账户**：各自初始资金/持仓/净值，summary 聚合视图（无共享现金池）。
- 作业状态：`queued | running | completed | failed | cancelled | interrupted`。

### 拒单原因枚举（调试友好）

`t_plus_1_not_sellable` · `limit_up_buy_blocked` · `limit_down_sell_blocked` · `suspended` · `zero_volume` · `invalid_price_tick` · `invalid_lot_size` · `insufficient_cash` · `insufficient_position` · `volume_cap_below_lot`

### CLI（`vortex-backtest`）

```bash
vortex-backtest serve --host 127.0.0.1 --port 8766
vortex-backtest account create --id demo --cash 100000
vortex-backtest order add --account demo --request-id buy-1 --date 2026-01-02 --symbol 000001.SZ --side buy --qty 100 --batch batch-main
vortex-backtest backtest run --account demo --start 2026-01-02 --end 2026-01-05 --batch batch-main --wait
vortex-backtest report <job_id> --what summary|daily|trades|rejections
```

## 三、vortex_qmt（实盘交易）· http://127.0.0.1:8767

### 端点

| 方法 | 端点 | 写 | 说明 |
|---|---|---|---|
| GET | `/` · `/health` · `/status` | | 元信息/健康/连接与近期执行 |
| GET | `/account/cash` · `/nav` · `/capital` · `/positions` · `/orders` · `/fills` | | 资金/净值/策略资金/持仓/委托/成交 |
| GET | `/market/quotes?symbols=` | | 实时行情快照 |
| GET | `/executions` · `/executions/{exec_id}` | | 近期执行列表/回读执行报告 |
| POST | `/account/capital` | ✓ | 设定策略资金（持久化/可重置） |
| POST | `/orders` | ✓ | 单笔买卖 |
| POST | `/rebalance` | ✓ | **目标组合调仓全链路** |
| POST | `/reconcile/{exec_id}` | ✓ | 日终对账 |

### 目标组合调仓（请求）— 策略 → 实盘的核心契约

```json
{
  "trade_date": "2026-06-08",
  "strategy_version": "momentum_v1",
  "run_id": "run-20260608",
  "snapshot_id": "snap_20260605_220000",
  "notional": 1000000,
  "positions": [
    {"symbol": "000001.SZ", "target_weight": 0.10, "reference_price": 11.80, "reason": "减仓"},
    {"symbol": "600519.SH", "target_weight": 0.20, "reference_price": 1650.0}
  ],
  "st_flags": {"000001.SZ": false},
  "enable_trading": false,
  "account_id": null,
  "confirmation": null
}
```

响应含 `exec_id`、`report`（order_plan / risk_result / cash / slippage / lineage）、`artifacts`（落盘路径）。`mode` 可为 `qmt_dry_run`（默认只算不发）/ `qmt_sim`（模拟/真实）/ `paper`（本机撮合）。

### 实盘三重门禁（缺一不可）

1. `enable_trading=true`（且全局总开关 `VORTEX_QMT_ENABLE_TRADING=true`）
2. `account_id` 命中 `VORTEX_QMT_ALLOWED_ACCOUNT_IDS` 白名单
3. `confirmation == "CONFIRM_AUTO_TRADING"`

### 事前 fail-closed 风控（12 规则，任一不过或数据缺失 ⇒ 整批零下单）

`broker_health` · `mode_allowed` · `order_count` · `cash_non_negative` · `quote_price_available` · `single_order_value` · `daily_order_value` · `not_suspended` · `limit_up_buy` / `limit_down_sell` · `lot_size` · `limit_price` · `not_st_buy`(+`st_data_available`)

### 错误码

`400 missing_symbols` · `401 token` · `403 live_trading_denied` · `404 execution_not_found` · `404 missing_quotes` · `422 invalid_target_portfolio`

### 委托状态映射（内部 ← qmt-bridge 码）

`open←50` · `partial←含"部分"` · `filled←56` · `cancelled←53/54` · `rejected←55/57`

### CLI（`vortex-qmt`）

```bash
vortex-qmt serve     # http://127.0.0.1:8767
vortex-qmt demo      # 冒烟
vortex-qmt config    # 查看配置
```
