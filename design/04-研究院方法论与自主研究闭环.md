# 04 · 研究院方法论与自主研究闭环

> 注：配置/端口以 vortex_common ADR-003 + config/registry.yml 为准（本文为历史记录）。

本文件定义 vortex_trader 作为"量化研究院／交易员"如何运转：研究闭环 SOP、agent 研究员分工、当前数据资产、研究方向建议，以及"先定方法论、再自动化"的演进路线。

## 一、研究闭环（八步 SOP）

每一个策略想法都走同一条可复现、可审计的闭环。括号内是该步对接的服务/接口。

1. **假设（Hypothesis）**：用一句话写清因子/策略的经济学逻辑与预期方向，登记到研究记录。
2. **取数（Data）**：在 vortex_data 确认所需数据集已落盘、覆盖区间足够（`GET /api/datasets`、`/api/datasets/{ds}/rows`，或直接读 `workspace/qlib`）。缺数据 → 提抓取需求（`POST /api/data/task`）。
3. **构建（Construct）**：计算因子/信号，落成可回放的**订单批**或**目标组合**（symbol / 方向 / 数量或权重）。
4. **回测（Backtest）**：提交 vortex_backtest（`POST /backtests`），轮询 `job_id`，取日级 `summary/daily/trades/rejections`。
5. **评估（Evaluate）**：看收益/回撤/Sharpe/换手、与基准对比、拒单分布；做稳健性检验（见第四节标准）。
6. **纸上/模拟（Paper/Sim）**：通过则在 vortex_qmt 以 `qmt_dry_run` / `paper` 验证目标组合能生成合法订单计划、过风控（`POST /rebalance`，`enable_trading=false`）。
7. **实盘（Live）**：多交易日模拟盘稳定后，三重门禁人工确认上线（`enable_trading=true` + 账户白名单 + `CONFIRM_AUTO_TRADING`）。
8. **归档（Archive）**：把假设、参数、回测 `job_id`、数据 `snapshot_id`、执行 `exec_id` 一并存档，保证结论可复现、可追溯。

> 关键纪律：**执行层只消费冻结好的目标组合，不临场重算信号**；A 股 T+1 由策略侧建模（信号日 T → 次日执行）。

## 二、研究院的 agent 分工

按"用 agent 并行推进"的思路，把研究院拆成可由独立 agent 承担的角色（职责单一、产出可汇总）：

| 角色 | 职责 | 主要对接 |
|---|---|---|
| **首席（PI，本人/主控）** | 选题、汇总、决策、文档沉淀 | 统筹全局 |
| **数据官（Data Steward）** | 监控数据覆盖/质量，提补数据与补历史需求 | vortex_data API / manifest |
| **因子研究员 ×N（并行）** | 各探索一个因子族（动量/反转/资金流/筹码…），出因子定义与初步回测 | data 取数 + backtest |
| **回测工程师（Backtest Eng.）** | 标准化回测提交、参数扫描、指标采集 | vortex_backtest API |
| **风控/验证员（Validation）** | 稳健性与过拟合审查、对抗式质疑 | 评估标准（第四节） |
| **交易员（Trader）** | 策略→目标组合编排、纸上/实盘执行与对账 | vortex_qmt API |

并行方式：因子研究员可作为多个并发 agent 同时跑不同因子族（fan-out），首席汇总进排行榜后再决定哪些进入纸上/实盘。

## 三、当前数据资产盘点（决定能做什么）

来源：vortex_data 注册表（51 数据集）+ `workspace` 实际落盘 + qlib 导出盘点。

**已就绪（立即可用）**：
- 行情基础：`bars`(日线)、`adj_factor`(复权)、`valuation`(估值)、`stk_mins`(分钟，2026 年)——覆盖完整。
- 风控数据：`stk_limit`(涨跌停)、`suspend_d`(停复牌)、`limit_list_d`——完整。
- 资金/筹码/龙虎榜：`moneyflow`(个股/北向/板块/大盘)、`cyq_perf`/`cyq_chips`、`top_list/top_inst`——完整。
- 指数/行业/概念：`index_daily`、`index_weight`、`sw_daily`、`ths/dc_index`——完整（基准与行业轮动可用）。
- **Qlib 导出**：`cn_day` 5525 标的 × 约 100 交易日（2026-01-05~06-05）；`cn_1min` 5525 标的 × 约 1 个月；均含 12 字段（含 `factor`/`limit_up`/`paused`）。

**关键缺口**：
- 财务：有资产负债表、现金流量表；**缺利润表 `fundamental`、财务指标 `fina_indicator`、业绩预告/快报 `forecast`/`express`**。
- 历史跨度短：日线约 100 个交易日、分钟约 1 个月——**参数时间稳健性难验证**。
- 宏观/新闻：部分缺（cpi/ppi/m、新闻语料全缺）。

## 四、研究方向建议（你全选了，给出排序与依据）

| 方向 | 可行性 | 依据 | 结论 |
|---|---|---|---|
| **量价技术因子** | ✅ 立即可做（9/10） | 行情/复权/涨跌停/资金流/筹码/分钟全齐 | **先做**：动量、反转、量价背离、资金面、筹码胜率、行业轮动 |
| **多因子选股** | ⚠️ 部分（6/10） | 技术与质量(资产负债)可用，盈利/成长/价值因子缺基本面 | **并行起步**：先纯技术+行业轮动多因子，待基本面补齐扩展 |
| **基本面/财报驱动** | ⚠️ 需补数据（5/10） | 缺利润表/财务指标/预告 | **先补数据**：抓 `fundamental`(P1)、`fina_indicator`(P2)、`forecast/express`(P3) |

**推荐节奏**：以量价因子立刻建立研究闭环原型（验证 data→backtest→评估全链路）；数据官并行补利润表/财务指标与**补历史跨度**；待基本面与更长历史就绪，再融合成多因子选股组合。

> 注意：当前样本仅约 100 个交易日，任何回测结论都应标注"样本极短、待更长历史复核"，避免过早下定论。

## 五、评估与稳健性标准（防过拟合）

- **核心指标**：年化收益、最大回撤、Sharpe/Sortino/Calmar、换手率、与基准（沪深300 等）的超额。
- **稳健性必查**：样本外/分段表现、分板块（主板/科创/创业/北交所）表现、参数邻域稳定性（非单点最优）、交易成本敏感性。
- **A 股现实约束**：拒单分布要合理（涨跌停/停牌/T+1 导致的不可成交不能被忽略）；换手过高会被费用吃掉。
- **对抗式验证**：由验证员 agent 专门质疑——"这是不是只在这 100 天有效？""换费率/换板块还成立吗？"

## 六、血统与可复现

每条结论都应能凭三个 ID 复现：
- `snapshot_id`（vortex_data 导出快照，锁定数据 as_of）
- `job_id`（vortex_backtest 回测作业）
- `exec_id`（vortex_qmt 执行，确定性派生自 portfolio_id+trade_date+订单序列）

同一 `snapshot_id` 上回测 100% 可重复；A/B 对比必须同快照。

## 七、自动化路线（先定方法论，再自动化）

按你选定的"先定方法论再自动化"，分三步走，**当前停在第 1 步**：

1. **手工跑通闭环（现在）**：用上面的 SOP 人工/agent 跑通至少一个量价因子的完整闭环，确认每个接缝可用、口径正确。
2. **固化为脚本/SOP**：把"取数→构建→回测→评估→出报告"沉淀成可重复脚本与清单（本库 `design/` + 后续 `scripts/`）。
3. **再上自动化**：成熟后用定时任务每个交易日自动"取数→跑既定因子→回测→生成研究日报"；实盘仍保留人工门禁，不自动下真实单。

## 八、近期里程碑

- **M1（已完成）**：四库调研、端到端契约、数据盘点、本套枢纽文档与架构图。
- **M2**：跑通端到端——启动三服务，用量价因子做一个 baseline 回测，验证 qlib/目标组合契约（含把 backtest 主路径对齐到 qlib 的评估）。
- **M3**：建"编排桥"——把回测/策略产出自动转成 vortex_qmt `/rebalance` 目标组合，打通 dry-run 全链路。
- **M4**：数据补强——补利润表/财务指标 + 补历史跨度；扩展到多因子选股。
- **M5**：统一技术文档定稿 + 实盘上线 SOP（多日模拟盘 → 三重门禁上线）。
